Module 5: Objects and Classes

The StringBuffer class is located in the java.lang package. A major difference between the String class and the StringBuffer class is found in the flexibility of the latter. Once a String object’s value has been created, it cannot be changed. StringBuffer, however, has the ability to change the contents by adding, inserting, or appending new material into the string buffer, as shown below.

StringBuffer sb = new StringBuffer();
sb.append("Hello");   // inserting a string (Hello)
sb.append(' ');       // appending a character space
sb.append("World.");  // appending a string (World.)

Suppose we would like to change the period after World to an exclamation point. After Hello World. has been put into the object, sb, the period is in the 11th position. Because the index runs from 0 to 11 in this string, we use the following code fragment to delete the period:

sb.deleteCharAt(11);  // deletes the period in the 11th position

The result is that the string is now Hello World and we can append the exclamation point correctly at the end:

sb.append("!");  // appends the exclamation point

Now the string contains:

Hello World!

There are many other methods found in StringBuffer that allow us to manipulate strings.

When JDK 1.5 became available, the StringBuilder class was introduced. It is almost the same as StringBuffer and so may be used in its place. StringBuilder is normally used when it is to be accessed by a single task. StringBuffer can be accessed by multiple tasks.
