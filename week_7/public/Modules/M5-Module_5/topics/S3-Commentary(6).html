Module 5: Objects and Classes

A. The String Class

In Java, there is a class named String, which implements strings of characters, such as names and addresses. We have seen in previous sections the way to declare and create an object:

String address = new String("ABC");

One way to create two strings, s1 and s2, which contain the values ABC and DEF, is shown in the following code fragment.

String s1 = new String("ABC");
String s2 = new String("DEF");

But we do not always have to use the new operator with a String.

One more thing to note is the difference between Strings and chars. A char is a primitive and exists in the Java language itself. It is a single data value (a single char) of fixed size. In Java, when a char is used as a literal, it is enclosed in single quotes. For example:

the literal 'A' is a char

A String is an object that is made up of one or more chars. Even if the String has only one char in it, the String is still an object and has the behavior of objects that will be defined in the next section. In Java, when a String is used as a literal, it is enclosed in double quotes. For example:

the literal "A" is a String

In II-D, we discussed object methods. As with class methods, it is often necessary to state explicitly which object the method is to use. Once again, the . (dot) operator is used to specify the particular object that is to be used for the method. Just as with class methods, the left-hand side of the . (dot) operator is the object to use, and the right-hand side is the method to be called. The following code fragment illustrates this point:

String sb1 = new String("ABC");
String sb2 = new String("DEFGHI");
System.out.println("The third character of sb1 is " + sb1.charAt(2));
System.out.println("The third character of sb2 is " + sb2.charAt(2));

In the code above, the charAt method of the String class is run twice, and in each case the third character is taken from the String. (Note that the index for the String ranges from 0 to length() – 1. The first char in the String is at index 0, the next at index 1, and so on.)

In the first println statement, the method charAt, which is on the right-hand side of the . (dot) operator, retrieves the third character from the String sb1, which is on the left-hand side of the . (dot) operator. Because sb1 contains the String ABC, the third character, C, is printed.

In the second println statement, the method charAt, which is on the right-hand side of the . (dot) operator, retrieves the third character from the String sb2, which is on the left-hand side of the . (dot) operator. Because sb2 contains the String ABC, the third character, F, is printed.

To use an instance method, it is necessary to first allocate an object to provide a context for that method. For example, consider the following code fragment:

String sb1;
System.out.println("The third character of sb1 is "
                    + sb1.charAt(3));  // Invalid

This code fragment does not make sense because there is no String to query for the third character. This situation is very different from trying to access a character outside of the range of the String. Attempting to call a method on an object that has not been initialized never makes sense and will always result in a null reference exception. Because exceptions will not be covered in this course, you may consider an exception to be an error message.

Some examples of using String methods are presented in the following program. Move your mouse over OUTPUT to see what is printed.

/* File:    UMUC_ShowString.java
 * Author:  InstructorX
 * Date:    dd-mm-yyyy
 * Purpose: Show some string functions
 */

public class UMUC_ShowString {
   public static void main(String args[]) {
      String s1 = new String("ABCDEF");
      String s2 = new String("UVWXYZ");

      // c1 is set to D, the character in
      // s1 at position 3
      char c1 = s1.charAt(3);
      System.out.println("c = " + c1); // OUTPUT

      // c1 is set to X, the character in
      // s2 at position 3
      c1 = s2.charAt(3);
      System.out.println("c = " + c1); // OUTPUT
          
      // s3 is set to CDE, a substring of s1 from
      // positions 2 to 5
      String s3 = s1.substring(2,5);
      System.out.println(s3);          // OUTPUT
      
   } // end method
} // end class

A complete catalog of all String methods can be found in the Java API located at java.sun.com.

We will now discuss one other important method in the String class—the equals method. Recall that the == operator is used in Java to compare two objects to see if they are the same. This comparison is called equality.

There are times, however, when what is to be checked is not that the objects are the same, but that the values contained in the objects are the same. If the values in the two objects are the same, it is said that the objects are equivalent. For example, consider the following program:

/* File:     UMUC_TestString.java
 * Author:   InstructorX
 * Date:     dd-mm-yyyy
 * Purpose:  Test two strings for equality and equivalence
 */
   
public class UMUC_TestString {
   public static void main(String[] args) {
      String s1 = new String("ABC");
      String s2 = s1;
      String s3 = new String("ABC");

      if (s1 == s2)
         System.out.println("s1 and s2 are equal");      // OUTPUT
      if (s1.equals(s2))
         System.out.println("s1 and s2 are equivalent"); // OUTPUT
      if (s1 == s3)
         System.out.println("s1 and s3 are equal");
      if (s1.equals(s3))
         System.out.println("s1 and s3 are equivalent"); // OUTPUT
   } // end method
} // end class

In this program, Strings s1 and s2 are equal (they refer to the same object). Because they refer to the same object, that object must have the same value, so they are also equivalent.

Strings s1 and s3 refer to different objects, so they are not equal. The Strings s1 and s3 have the same data, however, so these Strings are equivalent.

In summation:

  • If two variables represent the same object, they are equal and equivalent.

  • If two variables represent different objects that have the same values, they are not equal but are equivalent.

  • If two variables represent different objects that have different values, they are neither equal nor equivalent.

B. The Character Class

In I-D, we introduced wrapper classes found in the java.lang package. Although the two examples applied to only doubles and integers, there are wrapper classes for all of the primitive data types. The Character class has several methods that allow the manipulation of characters. For example, we can compare two characters and determine if they are equal.

Character char1 = new Character('a');
Character char2 = new Character('b');
char1.compareTo(char2);    // returns 1 because they are 1 unit apart
char1.equals(char2);       // returns false because they are not equal

Other methods will allow us to determine if the character is a letter or a digit, or if it is uppercase or lowercase. Two methods will return either a lowercase or an uppercase character. If the Character class is used with the String class, it is possible to conduct several operations on the contents of the string.
