Using new with String Objects

Often, the new operator is not used when allocating a String object. For example, when initializing a String, we may write:

String s = "ABC"

instead of writing:

String s = new String("ABC");

The reason that not using the new operator works is because the String object is a special object, unique in Java. When the language was implemented, the designers decided to allow programmers to specify String objects, and String objects only, as a literal string of characters. Behind the scenes, Java is actually creating an object for you using the new operator. To emphasize the fact that an object is always created, however, this module will always use a new operator when creating a String object.
