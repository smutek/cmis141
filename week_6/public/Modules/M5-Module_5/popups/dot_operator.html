Which Instance of the Class to Use:
Objects and the Dot ( . ) Operator

You may be wondering how Java knows which object to associate with a given instance method. In module 3, we pointed out that an object must always be called with the dot operator, where the left-hand side of the operator was an object. That object on the left-hand side of the dot operator is the one whose instance variables will be used in the current call to an instance method.

Therefore, when calling the method InstanceMethod, the programmer must specify the object to use, for example:

UMUC_Variables v1 = new UMUC_Variables();
v1.InstanceMethod();
