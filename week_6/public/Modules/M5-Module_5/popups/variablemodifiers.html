VariableModifiers

As with local variables and methods, modifiers can also be associated with a variable associated with a class. These modifiers are static, final, public, private, and protected. The use of the modifier static makes the variable a class variable, not an instance variable.

Static variables are never instance variables, so static should never be used as a modifier for instance variables. Also, the protected modifier has meaning only when inheritance is used, so we will not consider it in this course.

We have already discussed the final modifier, which gives a variable a fixed value. The variable cannot be changed if it is used with a primitive that is an instance variable, so a final variable will be effectively the same for every object. Because there is no ability to store data with each object using a final variable that is a primitive, the final modifier should never be used with an instance variable that is a primitive.

There are uses of the final modifier for object instance variables, but these are related to how the Java VM handles some specific issues that are outside the scope of this course. So the use of the final modifier will not be considered with instance variables.

Therefore, the only modifiers that will be used in this class with instance variables are private and public. As we mentioned earlier, it is considered bad form to declare a variable as anything but private.
