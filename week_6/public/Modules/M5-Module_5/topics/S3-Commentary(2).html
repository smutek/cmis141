Module 5: Objects and Classes

One of the most powerful features of Java is not simply that it enables programmers to use predefined classes, but that it allows them to develop classes to solve problems they encounter. In this section, we will continue to describe the way to create and use classes to solve a problem.

Instance Variables

Instance variables are used to define the data that make up an object that is an instance of a class. Instance variables are defined as part of the class and are declared inside the class block, but outside of any method. Instance variables can be of any type, including primitives and other objects, and are similar to local variables, which were discussed in section I of this module.

There is one difference between instance variables and local variables. Local variables are confined to the method in which they are declared and cannot be accessed outside of that method. Instance variables can always be accessed in any method in the class in which they are defined, whether they are declared public or private.

It is also possible to access instance variables outside of the class in which they are defined by using the modifier public when the variable is declared. If an instance variable is declared private, it can be used only in the class in which it is defined.

Encapsulation

Declaring instance variables private and allowing access to them only through methods is called encapsulation because the variables are encapsulated in the object, and access can be restricted.

Encapsulation is a powerful technique to help ensure a program is correct because all access can be controlled through methods. Things like bounds checking on the variables can thus be enforced in a program.

Later in this module, we will discuss some reasons for using public variables that are associated with a class.

Methods

Instance methods and class methods are defined in the same way, with three exceptions:

  1. Class methods are normally declared public static, and instance methods are normally declared simply public. We will discuss the reason for this difference later.

  2. Instance methods use and change instance data for a specific object. The object that is operated on is the one that was used with the dot operator to call the method.

    In many cases, two special methods, called setters and getters, are created to set and get the values of the instance data. These methods are generally named with set or get, followed by the name of the variable they access, although using set or get is not required. Some examples are:

    • setPayRate and getPayRate for the variable payRate
    • setHoursWorked and getHoursWorked for the variable hoursWorked

  3. There is one special type of class method, called a constructor, which we will now discuss.
Constructors

Constructors are a special type of instance method. They differ from normal methods in the following ways:

  • Constructors must be named the same as the class name.

  • Constructors do not have a return type.

  • Because constructors do not have a return type, no return statement is needed at the end of the method.

  • Constructors are not explicitly called but are invoked when an object is created using the new operator.

All classes that are used to instantiate objects must have at least one constructor. For this module, the only type of constructor to be considered is a default constructor, which has no parameters. This constructor will be included in all examples of programs implemented for this module.

B. A Simple Class Definition

A class is a much larger and more complicated structure than any dealt with so far, but if we explain it step by step, it is not too hard to understand. First, we will list the steps, and then we will give an example to clarify them. The steps, in order, are:

  1. Build a class block, which contains the class declaration and braces to surround the definition of the class.

  2. Declare any data needed for the class.

  3. Declare and implement the constructors for the class. In this module, only the default constructor is used, so its declaration is simple.

  4. Define and implement any methods needed for the object. Specifically, you may need to define a setter and getter for each data item in the class. Also, you must define any other methods needed to implement behavior the class needs.

  5. Create a main method to test that what you have done is correct.

This process looks more complicated than it actually is. The following section leads you through the steps needed to implement a simple object.

C. Implementing a Simple Object

Problem Definition

There is a small company with two employees, Tom and Jane. Each employee has:

  • a pay rate per hour that he or she works
  • a number of hours that he or she works

We wish to create an object for each employee that stores the pay rate and hours worked. We then want to call a method to calculate the total pay that each has earned.

Define the Class

From this problem, we can see that the two employees have similar characteristics: they each have a pay rate and a number of hours worked. Therefore, we can abstract out these characteristics to create a common Employee class that has two instances, one each for Tom and Jane.

The implementation of the base Employee class is shown in the code fragment below:

public class Employee {
} // end class
Define the Data

The next step is to define the data needed for this class. Two data fields are needed, one that stores the pay rate per hour and the other that stores the hours worked. After asking the user, we find that the pay rate per hour and the hours worked should each be stored as a double. We now add these two data fields to the class definition. The new part of the code is highlighted.

public class  Employee {
   private double payRate = 0.0;
   private double hoursWorked = 0.0;
} // end class
Implement the Default Constructor

The default constructor is now added to the class. Note that this is a method with no return type and no parameters, and that it has the same name as the class. In our constructor, we will set some default values for the two instance variables. Again, the new part of the code is highlighted.

public class Employee {
   private double payRate= 0.0;
   private double hoursWorked = 0.0;

   public Employee() {
    payRate = 20.54;
    hoursWorked = 40.0;
   } // end constructor
} // end class
Implement Methods for Behaviors Needed by the Objects

Now, methods are written that allow the object to have the behaviors needed in the problem. These methods typically use the instance variables defined in step 3. First, two methods are needed for each instance variable, one to set the value and one to get the value. These methods are called setters and getters in Java, and they normally have the string set or get pre-pended to their name. For example, to set the number of hours worked, the method setHoursWorked is defined.

In addition to the setters and getters, there is also a need for a method to calculate the pay for the employees. This method simply multiplies the payRate by the hoursWorked and returns the result.

The methods added to the object are shown in highlight.

public class Employee {
   private double payRate = 0.0;
   private double hoursWorked = 0.0;

   public Employee() {
      payRate = 20.54;
      hoursWorked = 40.0;
   } // end method

   public void setPayRate (double rate) {
    payRate = rate;
   } // end method

   public double getPayRate () {
    return payRate;
   } // end method

   public void setHoursWorked(double hours) {
    hoursWorked = hours;
   } // end method

   public int getHoursWorked() {
    return hoursWorked;
   } // end method


   public double calculatePay() {
    double pay = payRate * hoursWorked;
    return pay;
   } // end method
} // end class
Adding the main Method to Create a Program

Now a main method is added to the class to create the following program, shown below in highlight. Note that the main method creates an object for each Employee, sets the values for the variables, and then calls the calculatePay method to print out the amount earned by each employee.

D. Using Instance Methods

As we stated earlier, you cannot use operators on objects, so to access and change an object you must use methods that are provided with the object's class. The methods used to access and change an object are similar to the class methods discussed earlier. Methods used on objects, however, do not behave the same way every time they are called. Rather, they are dependent on the value of the data that make up the object. Because an object is an instance of a class, these methods are often called instance methods.

As with class methods, it is often necessary to state explicitly which object the method is to use. Once again, the . (dot) operator is used to specify the particular object that is to be used for the method. Just as with class methods, the left-hand side of the . (dot) operator is the object to use, and the right-hand side is the method to be called. We created two instances of the class Employee earlier, jane and tom. The data fields were hoursWorked and payRate. The following code fragment declares and creates the two objects:

Employee jane = new Employee();
Employee tom = new Employee();

Suppose we want to print the rate of pay for Jane. The following code fragment is an example of the way to reference a data field for an object:

System.out.println(jane.payRate);

If we wanted to print out the pay for Tom, we would use the following code fragment, which shows the way a method for an object can be referenced:

System.out.println(tom.calculatePay());
