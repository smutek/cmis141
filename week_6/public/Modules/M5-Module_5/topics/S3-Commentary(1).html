Module 5: Objects and Classes

A. Object Overview

One example of using an object would be a program to determine an employee's pay, where that pay is calculated by multiplying the number of hours worked by the hourly pay rate. For the employee object, there would need to be two data fields—one for the hours worked and one for the pay rate per hour. There would also need to be behaviors that would give values to each of the data fields, as well as a behavior that calculates the pay for that employee. This scenario is represented by the following pseudocode:

Object Employee definition
   Has data fields of
      Rate of pay
      Hours worked
   Has behaviors of
      Set rate of pay
      Set hours worked
      Calculate Pay

Using the pseudocode above, we can create the Employee object. The class name would be Employee. Its data fields would be payRate and hoursWorked. Both can be of the type double. The behavior for the class will be defined by using methods to setPayRate, setHoursWorked, and calculatePay. Additional information will be given about the creation of classes and the constructors (Employee() and Employee(payRate: double, hoursWorked: double) in section II-C of this module.

B. Creating Objects and the new Operator

To create an instance of a class, or an object, in Java, the new operator is used. The new operator does two things:

  1. It allocates space in which to store the data that will make up the Java object.
  2. It then calls a special method, called a constructor, which initializes the data.

We will illustrate the use of the new operator using Employee as we create an instance of it called employee1.

Move your mouse over the highlighted lines to read the explanations.

Employee employee1;
 employee1 = new Employee(6.50, 12.5);

We also can combine the two previous lines into one line:

Employee employee1 = new Employee(6.50, 12.5);

C. Objects Are Defined by a Class

As we have seen in the previous section, one use of a class in Java is to create a type to define a variable. Because using a new operation involves creating an instance of Employee, it is sometimes called instantiating the object, and the object is called an instance of the class. Because an object is an instance of a class, the terms object and instance are used interchangeably in Java.

We saw that on some occasions when a variable for an object is declared, the object that corresponds to that variable is also created. Thus, one way to create two objects, employee1 and employee2, which contain the values 6.50 and 12.5 in the first object and the values 8.00 and 17.50 in the second object, is shown in the following code fragment.

Employee employee1 = new Employee(6.50, 12.5);
Employee employee2 = new Employee(8.00, 17.5);

Later, we will make use of this concept to create a program.

D. Differences between Objects and Primitives

Not all variables in Java are primitives. As we just discussed, some variables are objects. In our example, we designed a class for Employee and then created the object variables employee1 and employee2. There are other object variables that are existing classes. These types are not defined in the core Java language and are not Java reserved words, but are defined in a Java class. Therefore, programmers can use existing classes or create their own to define types for variables. In fact, in Java it is common for programmers to define their own classes for variables to be used in their programs.

Examples of object variables that are not primitive are String, Integer, Double, and Boolean. In Java, you can generally tell if a variable is an object or a primitive by the case of the first letter of the type. For example, an int is a primitive (because the first letter of the type is lowercase), but an Integer is an object (because the first letter of the type is uppercase). This convention is a coding standard that most Java programmers follow.

Also, every primitive in Java (e.g., int and boolean) has a class associated with it (e.g., Integer and Boolean) called a wrapper class. A wrapper class is used to store behaviors about that type of object, such as converting a String to int, as with the parseInt method used in the following code fragment:

anIntNum = Integer.parseInt("89");

The parseInt method converts a String into an int value. Also, the parseInt method is in the wrapper class Integer, as shown by the placement of the Integer class on the left-hand side of the dot operator, as we explained earlier.

For double conversion, the parseDouble method is available. The following code fragment shows how it might be used to convert a String into a double value:

aDoubleNum = Double.parseDouble("89.5");
